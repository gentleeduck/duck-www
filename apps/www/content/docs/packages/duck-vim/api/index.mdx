## Modules

duck-vim is split into independent modules. Each module has a single responsibility and can be imported separately.

| Module | Import path | Description |
| --- | --- | --- |
| [Platform](/docs/packages/duck-vim/api/platform) | `@gentleduck/vim/platform` | OS detection and Mod key resolution |
| [Parser](/docs/packages/duck-vim/api/parser) | `@gentleduck/vim/parser` | Parse, normalize, and validate key binding strings |
| [Matcher](/docs/packages/duck-vim/api/matcher) | `@gentleduck/vim/matcher` | Match keyboard events against parsed bindings |
| [Command](/docs/packages/duck-vim/api/command) | `@gentleduck/vim/command` | Registry and KeyHandler for managing shortcuts |
| [Sequence](/docs/packages/duck-vim/api/sequence) | `@gentleduck/vim/sequence` | Multi-step sequence matching |
| [Recorder](/docs/packages/duck-vim/api/recorder) | `@gentleduck/vim/recorder` | Record key combinations for settings UIs |
| [Format](/docs/packages/duck-vim/api/format) | `@gentleduck/vim/format` | Format bindings for display |
| [React](/docs/packages/duck-vim/api/react) | `@gentleduck/vim/react` | Provider, hooks, and context |

<Callout icon={<Lightbulb />}>

All modules except **React** are framework-agnostic and work in any browser environment.

</Callout>

<Callout icon={<MapIcon />}>

**Quick navigation:** Start with [Platform](/docs/packages/duck-vim/api/platform) for OS detection, then [Parser](/docs/packages/duck-vim/api/parser) and [Matcher](/docs/packages/duck-vim/api/matcher) for core key binding logic. Use [Command](/docs/packages/duck-vim/api/command) for the full registry system or jump straight to [React](/docs/packages/duck-vim/api/react) for hook-based usage.
